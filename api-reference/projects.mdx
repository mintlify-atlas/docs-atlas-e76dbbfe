<Note>
  The `rate` field is returned as a string in JSON to preserve decimal precision. Parse it with a decimal library rather than a native float.
</Note>

## List projects

Returns a paginated list of projects in your organization, ordered newest first. Unlike invoices and clients, archived projects are **not** automatically excluded — use the `status` filter to narrow results.

```
GET /api/v1/projects
```

### Query parameters

<ParamField query="page" type="number" default="1">
  Page number to retrieve. Must be a positive integer. Defaults to `1` if missing or invalid.
</ParamField>

<ParamField query="per_page" type="number" default="20">
  Number of projects per page. Minimum `1`, maximum `100`. Defaults to `20` if missing or invalid.
</ParamField>

<ParamField query="status" type="string">
  Filter projects by status. Must be one of: `ACTIVE`, `COMPLETED`, `ARCHIVED`. Returns a `400` error if the value is not a valid status.
</ParamField>

### Example requests

```bash
# All projects, newest first
curl https://invoices.example.com/api/v1/projects \
  --header "Authorization: Bearer <your-access-token>"

# Only active projects
curl "https://invoices.example.com/api/v1/projects?status=ACTIVE" \
  --header "Authorization: Bearer <your-access-token>"
```

## Get a project

Returns a single project by ID, including the client summary and all project tasks ordered by creation date.

```
GET /api/v1/projects/{id}
```

### Path parameters

<ParamField path="id" type="string" required>
  The project's unique ID (cuid).
</ParamField>

### Example request

```bash
curl https://invoices.example.com/api/v1/projects/clxproj1a2b3c4d5e6f7g8h \
  --header "Authorization: Bearer <your-access-token>"
```

### Response fields

The single project response includes all fields from the list response, plus:

<ResponseField name="data" type="object">
  The full project object.

  <Expandable title="properties">
    <ResponseField name="tasks" type="object[]">
      All tasks for this project, ordered by creation date (oldest first).

      <Expandable title="properties">
        <ResponseField name="id" type="string">
          Task ID.
        </ResponseField>
        <ResponseField name="name" type="string">
          Task name.
        </ResponseField>
        <ResponseField name="notes" type="string | null">
          Optional task notes.
        </ResponseField>
        <ResponseField name="sortOrder" type="number">
          Display order within the project.
        </ResponseField>
        <ResponseField name="isCompleted" type="boolean">
          Whether the task has been marked complete.
        </ResponseField>
        <ResponseField name="projectedHours" type="number">
          Estimated hours for this task.
        </ResponseField>
        <ResponseField name="rate" type="string">
          Task-level rate as a decimal string. Overrides the project rate when billing this task.
        </ResponseField>
        <ResponseField name="isFlatRate" type="boolean">
          Whether this task is billed at a flat rate.
        </ResponseField>
        <ResponseField name="dueDate" type="string | null">
          ISO 8601 due date for this task.
        </ResponseField>
        <ResponseField name="createdAt" type="string">
          ISO 8601 datetime when the task was created.
        </ResponseField>
        <ResponseField name="updatedAt" type="string">
          ISO 8601 datetime when the task was last modified.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

Returns `404` with `{ "error": "Not found" }` if the project does not exist or belongs to a different organization.
